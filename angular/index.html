模板式表单

只能使用指令定义模型NgForm NgModel NgModelGroup
使用NgForm用来代表整个表单，只要用angular起服务，会自动添加到每个form标签上，原form标签被ng接管，这时，在原form标签中的 form action=”/regist” method=”post”等 都不管用了。
NgForm 隐式创建了FormGroup类的实例，这个类用来代表表单的数据模型并且存储表单数据
标有ngform指令的html标签 如form会自动发现所有标有ngmodel指令的子元素并将它们的值添加到表单的数据模型中
ngform 可以在form标签之外的标签使用

ngModel 代表表单中的一个字段 隐式创建一个FormControl实例来代表字段的数据模型并用FormControl类型的对象来存储字段的值
注意 标记了ngForm指令的html元素内 使用ngModel指令不需要用[()]括起来 不需要把ngModel绑定在组件的某个属性， 要为添加了ngModel的元素加一个name

响应式表单

1、创建一个数据模型
用来保存表单数据的数据结构 由FormControl FormGroup FormArray组成
FormControl
FormGroup 多个FormControl集合 一个无效则FormGroup无效
整个表单或一个表单字段的固定子级

eg: formModel: FormGroup = new FormGroup({
form: new FormControl(),
to: new FormControl(),
})

FormArray 和FormGroup类似 多了一个长度属性
代表一个可以增长的字段集合，eg 允许用户输入任意数量的email
emails: FormArray = new FormArray({
new FormControl('XX@aaa.com),
new FormControl('XX@bbb.com),
})
没有key 只能通index访问

指令：

第二列 formGroup formControl使用属性绑定时使用的指令 FormArray不能通过属性绑定指令来绑定
第三列 可以使用属性名字连接数据模型和dom元素
响应式表单指令：form…
模板式表单： ng…
注意 1、响应式表单不可引用 不能用模板本地变量，不能通过创建模板本地变量引用指令的实例，不能拿到FormGroup等类。为了区分，模板式表单不能访问数据模型相关的类，只能拿到数据，可通过模板本地变量操作数据（只能在html中，不能在ts中）。响应式能访问类，不可引用，不能在模板中操作 （只能在ts中，不能在html中）
2、要在.module.ts中引入相应的module
模板式表单 FormsModule
响应式表单 ReactiveFormsModule

安全导航操作符 ( ?. ) 非空断言操作符（!）

安全导航操作符 当值为空时，保护视图渲染器，让它免于失败。

非空断言操作符（!）
在 TypeScript 2.0 中，你可以使用 –strictNullChecks 标志强制开启严格空值检查。TypeScript 就会确保不存在意料之外的 null 或 undefined。
在这种模式下，有类型的变量默认是不允许 null 或 undefined 值的，如果有未赋值的变量，或者试图把 null 或 undefined 赋值给不允许为空的变量，类型检查器就会抛出一个错误。
使用非空断言操作符 告诉类型检查器，它不会为空。
与安全导航操作符不同的是，非空断言操作符不会防止出现 null 或 undefined。 它只是告诉 TypeScript 的类型检查器对特定的属性表达式，不做 “严格空值检测”。
