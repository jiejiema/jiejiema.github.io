git 上点 New repository

1、（先进入项目文件夹）通过命令 git init 把这个目录变成git可以管理的仓库

2、把文件添加到版本库中，使用命令 git add .添加到暂存区里面去，不要忘记后面的小数点“.”，意为添加文件夹下的所有文件

3、用命令 git commit告诉Git，把文件提交到仓库。引号内为提交说明

将文件命名为类似模块名的名字 mongodb.js
const mongodb = require(‘./mongodb’);
如果全局有这个变量 会覆盖全局的

axios引用问题

在 main.ts中引用axios（import axios from ‘axios’），不能直接在xx.vue中使用，将vue改为vue的原型链，就可以解决这个问题，
大概是因为 vue认为axios是第三方，框架并未主动将其绑定到 Vue 实例，需要手动挂载到vue原型上
main.ts中加 Vue.prototype.$axioss = axios；
在xx.vue中： this.$axioss.post（）……

有些页面需要拿到用户信息，如username，后端存储user，
1、可以存到cookie里，
npm install cookie-parser –save

node中写var cookieParser = require(‘cookie-parser’)
app.use(cookieParser())
根据http协议，会自动带到浏览器cookies中，不安全，且cookie容量小

2、可以存到session里，
npm install express-session –save

var session = require(‘express-session’)
app.use(session({
secret: ‘imooc’
}))
需要的话res返回给前端，相对安全。

session：
只有session id放在cookie里，session data不是
express 1.5.0+ 不需要使用cookie-parser中间件来使session工作
如果cookie-parser和session之间的secret不同会出错
